# Canvas Fun with Pixel Art

_April 15, 2026_

I built a pixel art editor using nothing but the HTML Canvas API and TypeScript. No libraries, no frameworks, no `npm install pixel-art-editor`.

## Why

I wanted a tool that worked exactly how I wanted — symmetric drawing modes, crisp export at any scale, and no bloat. Every existing editor either required an account or had a toolbar I didn't need.

## The tricky parts

**Undo/redo** — storing the full grid on every stroke is wasteful. I used a command pattern that stores only the changed pixels and the previous color. For most operations this is a few hundred bytes instead of hundreds of kilobytes.

**High-DPI** — canvas needs `devicePixelRatio` scaling to avoid blurry pixels. Every coordinate needs to be scaled on input and output.

**Fill tool** — a naive flood fill on a large canvas can blow the stack. Iterative queue-based fill is simple and safe.

## What's next

Layers, animation frames, and maybe WebRTC collaboration if I'm feeling ambitious.
