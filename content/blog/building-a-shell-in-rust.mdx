# Building a Shell in Rust

_June 20, 2026_

A Unix shell is one of those things you use every day but never think about — until you try to write one.

I spent a month building `rusty-shell`, a minimal POSIX-like shell. Here's what surprised me.

## It's mostly signal handling

The core loop is simple: print prompt, read line, parse, fork, exec, wait. What makes it hard is process groups.

Every pipeline needs its own process group. The shell needs to be in the foreground process group of the terminal, but it needs to step aside when a job runs. If you get SIGTTOU handling wrong, the shell just freezes.

## Job control is a state machine

Background jobs, foreground jobs, stopped jobs — each has a lifecycle. `SIGTSTP` (Ctrl+Z) stops a job, `SIGCONT` resumes it. The shell tracks all of this with a table of process groups.

## What I'd do differently

- Start with the POSIX spec open from day one instead of guessing
- Write tests for the parser early — shell syntax is trickier than it looks
- Use `nix` crate helpers instead of raw `libc` calls where possible
